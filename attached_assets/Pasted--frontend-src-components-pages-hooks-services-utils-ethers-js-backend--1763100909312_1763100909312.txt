ğŸ” Authentication Flow (How MetaMask Login Works)

    User clicks Connect Wallet in React frontend.

    Frontend requests a nonce from:
    POST /api/auth/nonce

    MetaMask signs the nonce (signMessage via ethers.js).

    Frontend sends signed message to backend:
    POST /api/auth/verify

    Backend verifies signature & issues JWT.

    All protected routes require Authorization: Bearer <jwt>.

ğŸ“ File Upload Flow (IPFS)
Option A â€” Client-Side Upload (Recommended)

    User uploads file in dashboard.

    React app uploads file directly to IPFS using Web3.Storage.

    Web3.Storage returns a CID.

    React sends POST /api/files/create to backend with:

    cid, filename, size, metadata

    File record saved in MongoDB.

Option B â€” Backend Upload (Less preferred)

    React sends file to Express.

    Express uploads to IPFS using Web3.Storage token.

    CID saved to MongoDB.
